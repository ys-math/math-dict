# 数学用語のための IME 追加辞書 📐

[![License: CC0-1.0](https://img.shields.io/badge/License-CC0_1.0-lightgrey.svg)](LICENSE)

[English README](README.md)

## 概要 - Overview

日本語 IME は日常語の変換には強い一方で、数学の用語はほとんど変換できません。
「じゅんどうけいしゃぞう」も「べきれいげん」も候補に出ず、記号や数学者名に至っては手も足も出ません。
この辞書を導入すると、そうした数学用語を普通の日本語入力と同じように変換できるようになります。

現在 **1,076 語** が収録されています。**Google 日本語入力 / Mozc、macOS 日本語入力、
Microsoft IME、ATOK** の4種類の形式で配布しています。

## 変換例 - Examples

- `じゅんどうけいしゃぞう` → 準同型写像
- `きょくだいいである` → 極大イデアル
- `べきれいげん` → 冪零元
- `そうこほもろじー` → 層コホモロジー
- `ぜんしょう` → ∀ ／ `てんそる` → ⊗ ／ `あーる` → ℝ
- `がろあ` → ガロア ／ Galois（数学者名はカタカナとラテン文字の両方が候補に出ます）
- `うぇるでふぁいんど` → well-defined

分野別の収録内容は [`terms/`](terms/) から閲覧・検索できます。

## 辞書の導入方法 - Installation

[`dist/`](dist/) からお使いの IME 用のファイルをダウンロードしてください。
リポジトリのクローンやビルドは不要です。

### Google 日本語入力 / Mozc

1. [`dist/google/math_dict_ja.txt`](dist/google/math_dict_ja.txt) をダウンロードする
2. `辞書ツール` を開く
3. `管理` をクリック
4. `新規辞書にインポート` をクリック
5. `ファイルを選択` からダウンロードしたファイルを選ぶ
6. お好きな `辞書名` をつける
7. `インポート` をクリック

### macOS 日本語入力

1. [`dist/macos/math_dict_ja.plist`](dist/macos/math_dict_ja.plist) をダウンロードする
2. `システム設定` → `キーボード` → `ユーザ辞書` を開く
3. ダウンロードした plist ファイルをドラッグ＆ドロップする

### Microsoft IME

1. [`dist/msime/math_dict_ja.txt`](dist/msime/math_dict_ja.txt) をダウンロードする
2. `単語の登録` → `ユーザー辞書ツール` を開く
3. `ツール` → `テキストファイルからの登録` を選ぶ

### ATOK

1. [`dist/atok/math_dict_ja.txt`](dist/atok/math_dict_ja.txt) をダウンロードする
2. `辞書ユーティリティ` を開く
3. `ツール` → `ファイルから登録・削除` を選ぶ

| 動作確認の状況 |
|:---|
| 実際に IME へ取り込んで動作を確認できているのは **Google 日本語入力のみ** です。他の3形式については、文字コード・BOM・改行コード・品詞名・plist の構造といった形式面の検査しか行っていません。取り込みに失敗する場合はバグですので、[Issue](../../issues) を立てていただけると助かります。 |

### 収録の方針

登録してあるのは **正しい読み** であり、略語ではありません。
「数学的帰納法」の読みは `すうがくてききのうほう` であって、`きのう` ではありません。
短い読みで長い語を呼び出す方式は採用していません。日常語と衝突してしまい（`きのう` は「昨日」です）、
普段の日本語入力をかえって使いにくくするためです。

## カタカナ語について - A note on katakana

| ご注意 |
|:---|
| `こほもろじー` と入力して `コホモロジー` が出るのは、**この辞書のおかげではありません**。IME にはもともと、かな入力をそのままカタカナに変換する機能があります。`こほもろじー` は `コホモロジー` へ一対一に転写できるため、**スペースキーを押せば変換できます**。 |

辞書側の項目は IME が元から出すカタカナ候補と同じ文字列になるため、統合されて表に出てきません。
これに該当する語が **94 語**（うち 63 語は数学者名のカタカナ表記）あります。
実害はありませんが、実質的には機能していません。該当語は [REVIEW.md](REVIEW.md) に
「IMEが自前で変換できる」として一覧してあります。

辞書が実際に役に立つのは、IME が自力では変換できない語です。

- 漢字熟語（準同型写像、冪零元、極大イデアル）
- カタカナと漢字の組み合わせ（コホモロジー群、層コホモロジー）— 単純な転写にならないため
- 数学記号（∀、⊗、ℝ）
- 数学者名のラテン文字表記（Galois、Grothendieck）

残りの 982 語がこれにあたります。

## 辞書に語を追加する - Contributing

[`terms/`](terms/) の該当ファイルを編集してください。ここが唯一の原本で、`dist/` 以下はすべて
生成物です。

```
読み <TAB> 単語 <TAB> 品詞 <TAB> フラグ（任意）
じゅんどうけい	準同型	noun
```

- **品詞** には IME に依存しない値を使います（`noun`、`noun-suru`（する接続）、`noun-adj`（な接続）、`person`、`symbol`、`latin`）。各 IME の品詞体系への変換はビルド側で行います。
- **分野** はファイル名から決まるため、行ごとに書く必要はありません。
- **フラグ** は任意です。カンマ区切りで `common`（読みが日常語と衝突する）と `low-confidence`（読みに確認が必要）を指定できます。

編集したら、次を実行してください。

```sh
python3 validate.py   # 形式の検査と REVIEW.md の再生成
python3 build.py      # terms/ から dist/ を生成
```

どちらも CI で実行されます。加えて CI では、コミットされている `dist/` が `terms/` から生成される
内容と一致するかどうかも検査しています。生成物の更新漏れは、辞書を取り込んでも新しい語が入って
いないという形でしか表面化せず、気づくのが遅れるためです。

## 注意事項 - Caveats

この辞書の語は言語モデルが原案を作成し、機械的な検査を通したものです。
検査で検出できるのは形式的な誤り（不正な仮名、重複、カタカナ語と読みの不一致など）に限られ、
実際にそれらは多数検出されました。しかし、**形式は正しいが実在しない語**は検査では検出できません。

- 読みが誤っている場合は、その項目が変換候補に出てこないだけなので実害はありません。
- 一方、**実在しない語は問題なく変換され、そのまま文章に紛れ込みます。** こちらのほうが厄介です。

おかしな語を見つけた場合はバグですので、[Issue](../../issues) を立てるか、該当行を削除してください。

なお `validate.py` は「読みのモーラ数は単語の漢字数以上」という規則で略語を弾いていますが、
これは下限を見ているにすぎません。`きのう→数学的帰納法`（3 モーラ / 6 漢字）は検出できますが、
`こゆう→固有値`（3 モーラ / 3 漢字）は通過してしまいます。
CI が通ったことよりも、差分そのものを確認してください。

日常語と衝突しやすい語や、読みが未確認の語は [REVIEW.md](REVIEW.md) にまとめてあります。
